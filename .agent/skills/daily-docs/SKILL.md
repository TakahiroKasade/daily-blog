# Daily Documentation Skill

## 概述
自動生成每日技術文件，格式為 `yyyyMMdd_中文功能說明.md`，記錄當日開發進度、功能變更、技術決策。

## 文件存放位置
```
docs/
├── 20260117_用戶認證模組優化.md
├── 20260116_錢包功能實作.md
└── ...
```

## 使用方式

### 生成今日技術文件
```
請幫我生成今日技術文件
```

### 指定功能說明
```
請幫我生成今日技術文件: [功能說明]
```

## 文件模板

```markdown
# [yyyyMMdd] [中文功能說明]

## 📋 概述
簡要描述今日主要工作內容

## 🎯 完成項目
- [x] 項目 1
- [x] 項目 2
- [ ] 進行中項目

## 📁 異動檔案
| 檔案路徑 | 異動類型 | 說明 |
|---------|---------|-----|
| `src/main/java/...` | 新增/修改/刪除 | 變更說明 |

## 💡 技術決策
### 決策 1: [標題]
- **背景**: 為什麼需要這個決策
- **方案**: 選擇的解決方案
- **理由**: 為什麼選擇這個方案

## 🔧 程式碼變更重點

### [模組名稱]
```java
// 關鍵程式碼片段或說明
```

## 🐛 已知問題
- 問題描述 1
- 問題描述 2

## 📝 備註
其他需要記錄的事項

## 🔗 相關連結
- [相關 Issue](#)
- [參考文件](#)
```

## 自動化流程

1. **掃描 Git 變更**: 分析今日 commit 和修改的檔案
2. **分類整理**: 按照模組/功能分類異動項目
3. **生成文件**: 依照模板填充內容
4. **存檔命名**: 以 `yyyyMMdd_功能說明.md` 格式儲存

## 最佳實踐

1. **每日結束前執行**: 在一天工作結束時生成文件
2. **搭配 Git 使用**: 確保所有變更都有 commit
3. **補充細節**: 自動生成後可手動補充技術決策說明
4. **團隊共享**: 將文件 push 到共用 repository
