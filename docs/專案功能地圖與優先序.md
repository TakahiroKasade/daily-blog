# 專案功能地圖與優先序 (Project Feature Roadmap)

本文件根據系統架構藍圖，整理出實作功能的優先順序。
核心策略：**先求有 (功能) -> 再求穩 (資料庫/安全) -> 最後求炫 (互動/效能)**。

## 🚀 實作優先序清單

### **Phase 1: 內容管理核心 (Content Core)** ✅ 已完成
*   目標：讓部落格從「唯讀」變成「可讀寫」，並能永久儲存資料。
- [x] **新增文章功能 (Create Post)**：實作後台表單，讓你可以寫文章。
- [x] **編輯/刪除文章 (Edit/Delete)**：寫錯字要能改，不想要的測試文要能刪。
- [x] **PostgreSQL 遷移**：將 H2 換成 PostgreSQL，確保資料不會重啟就消失。

### **Phase 2: 安全與權限 (Security Guard)** ✅ 已完成
*   目標：保護後台，不讓路人隨便改你的文章。
- [x] **Spring Security 整合**：引入依賴，設定 SecurityConfig。
- [x] **認證機制 (Authentication)**：實作 `CustomUserDetailsService`，從資料庫讀取管理員帳號密碼。
- [x] **權限控管 (Authorization)**：前端 `sec:authorize` 控制按鈕顯示 (Login/Logout/Admin)。
- [x] **DataSeeder 管理員初始化**：啟動時自動建立 admin 帳號 (BCrypt 加密)。

### **Phase 3: 登入與註冊頁面 (Login & Register UI)** 🔴 目前進度
*   目標：自訂美觀的登入/註冊頁面，取代 Spring Security 內建的醜陋預設頁面。
- [ ] **自訂登入頁面 (Custom Login Page)**：建立 `login.html`，整合深色科技風格。
- [ ] **LoginController**：建立控制器，處理 `/login` 路由。
- [ ] **註冊功能 (User Registration)**：建立 `register.html` 註冊表單。
- [ ] **RegisterController**：處理 `/register` 路由與新使用者建立邏輯。
- [ ] **SecurityConfig 更新**：將 `/register` 加入公開清單 (permitAll)。
- [ ] **密碼驗證**：註冊時確認密碼 (Confirm Password) 與基本格式驗證。
- [ ] **角色分配**：新註冊的使用者預設角色為 `USER` (只有讀取權限)。

### **Phase 4: 互動式履歷 (Interactive Resume)** 🟡 特色
*   目標：用互動體驗取代死板的 PDF，讓面試官眼前一亮。
- [ ] **履歷資料庫設計**：建立 `RESUME_EXPERIENCES` 表 (Experience, Education)。
- [ ] **時間軸前端 (Timeline.js)**：設計前端頁面，動態讀取資料庫並渲染成時間軸。
- [ ] **技能標籤過濾**：點擊 "Java" 標籤，時間軸只亮起跟 Java 有關的經歷。

### **Phase 5: 技術儀表板 (Tech Dashboard)** 🟢 加分
*   目標：展示 Backend 整合能力 (API, 排程)。
- [ ] **專案指標資料庫**：建立 `PROJECT_METRICS` 表。
- [ ] **排程器 (Scheduler)**：寫一個 `@Scheduled` 程式，每小時去抓 GitHub API (Commit數) 和 遊戲 API (最高分)。
- [ ] **儀表板視覺化**：在首頁或作品集頁面，用圖表或數字卡片顯示這些數據。

### **Phase 6: 極致效能 (Performance)** 🔵 進階
*   目標：證明你懂「高併發」架構。
- [ ] **Redis 快取**：把「首頁文章列表」快取起來，讓回應速度降到 10ms 以下。
- [ ] **CI/CD 自動化**：設定 GitHub Actions，Code 推上去自動部署到 Render。

---
**使用說明**：
每當完成一個功能，請將 `[ ]` 改為 `[x]`，並 Commit 更新此文件。
